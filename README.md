# ЕКЦ: график работы на месяц

Небольшое веб-приложение на Python для составления месячного графика сотрудников Единого Контактного Центра.

Что умеет:

- загружать сотрудников, учебные ограничения, дополнительные пожелания и заявки на работу в выходные или праздники;
- автоматически сканировать папку `операторы/` и читать `.docx` в подпапках сотрудников;
- хранить справочник операторов в локальной SQLite-базе;
- хранить отпуска сотрудников в локальной SQLite-базе;
- хранить праздничные дни в локальной SQLite-базе;
- автоматически строить график по рабочим интервалам внутри дня;
- не назначать сотрудников на время учебы и других строгих ограничений;
- допускать к выходным и праздничным сменам только тех, кто сам подал заявку.

Базовые часы работы ЕКЦ в приложении:

- будние дни: `08:00-22:00`
- выходные и праздничные дни: `09:00-21:00`

## Стек

- `FastAPI`
- `Jinja2`
- `httpx`
- `SQLite`

## Запуск

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload
```

После запуска откройте [http://127.0.0.1:8000](http://127.0.0.1:8000).

## Файлы данных

### 1. Папка `операторы/`

Теперь приложение умеет работать без `employees.csv`, если у вас есть структура вида:

```text
операторы/
  Авдеева Эльвира/
    Доп.инфа по графику.docx
  Пирожков Роман/
    Доп.инфа по графику.docx
```

Что происходит:

- имя сотрудника берется из названия подпапки;
- все `.docx` в подпапке читаются автоматически;
- локальный парсер извлекает явные даты, диапазоны дат, учебные интервалы, отпуск и повторяющиеся ограничения по дням недели.

Важно:

- в автоимпорте сейчас читаются только `.docx`;
- изображения, `.pdf` и `.pptx` в папках сотрудников пока не распознаются автоматически.

### 2. `employees.csv`

Необязательный служебный файл для совместимости. В обычной работе через интерфейс он не нужен.
Норма часов теперь считается автоматически для каждого оператора:

- `норма = рабочие дни месяца × (8 × ставка)`
- праздничные дни исключаются из нормы
- вручную `norm_hours` в интерфейсе больше не задается

```csv
employee_id,name,max_hours,norm_hours,max_consecutive_days,rate
ivanova,Анна Иванова,168,152,5,1.0
```

Поддерживаемые поля:

- `name`
- `employee_id`
- `norm_hours` или `preferred_hours` для CSV-совместимости
- `max_hours`
- `max_consecutive_days`
- `rate`

Поддерживаемые ставки:

- норма часов считается как `рабочие дни месяца × (8 × ставка)`
- максимум подряд: `0.25 -> 4 часа`, `0.5 -> 6 часов`, `0.75 -> 8 часов`, `1.0 -> 8 часов`

### 3. `study_schedule.csv`

Учеба или другое обязательное отсутствие. Если время не указано, ограничение считается на весь день.

```csv
employee_name,date,start_time,end_time,note
Анна Иванова,2026-03-10,09:00,18:00,Очная сессия
```

### 4. `preferences.csv`

Дополнительные пожелания и ограничения.

Поддерживаемые `kind`:

- `unavailable`
- `prefer_off`
- `prefer_work`
- `study`

Пример:

```csv
employee_name,date,start_time,end_time,kind,strict,note
Игорь Петров,2026-03-07,00:00,23:59,prefer_off,false,Семейное мероприятие
```

### 5. `weekend_choices.csv`

Заявки сотрудников на работу в выходные и праздничные дни. Это ключевое правило: если заявки нет, сотрудник в такой день не назначается.

```csv
employee_name,date,shift_code,note
Павел Соколов,2026-03-08,,Может выйти в праздник
```

`shift_code` можно оставить пустым, если сотрудник согласен на любую выходную или праздничную смену в этот день.

## База данных операторов

Приложение создает локальную SQLite-базу в `data/ekc_scheduler.db`.

Что хранится:

- ФИО оператора
- норма часов
- ставка
- норма в рабочий день, вычисляемая из ставки
- максимум часов подряд, вычисляемый из ставки
- максимум подряд рабочих дней
- признак активности
- отпуска сотрудников с датой начала и окончания
- праздничные дни и их названия

Как используется база:

- новые сотрудники из `операторы/` и `employees.csv` автоматически добавляются в базу;
- при расчете графика приоритет отдается значениям из базы, если оператор уже есть в справочнике;
- ставка ограничивает максимум часов подряд;
- отпуска из базы автоматически превращаются в строгие ограничения и блокируют назначение сотрудника на все дни отпуска;
- праздничные дни из базы автоматически меняют тип дня на `holiday`;
- если не загружать `employees.csv`, можно строить график только по активным операторам из базы и Word-файлам из папки `операторы/`.

## Логика планировщика

- приложение само раскладывает день на стандартные рабочие интервалы и покрывает часы ЕКЦ: будни `08:00-22:00`, выходные и праздники `09:00-21:00`;
- для сотрудников типа `оператор` по умолчанию заложено буднее покрытие: `08:00-09:00` ровно 1 оператор, `09:00-18:00` оператор не должен оставаться один, `18:00-22:00` 1 или 2 оператора;
- сотрудники типа `только заявки` могут получать только рабочие интервалы внутри окна `09:00-18:00`, и для будних дней им не нужна отдельная заявка;
- в интервале `09:00-18:00` рядом с оператором должен быть еще хотя бы один сотрудник: либо второй `оператор`, либо `только заявки`;
- понедельник считается самым пиковым днем, и планировщик старается сильнее приоритизировать операторов в дневное время;
- сотрудники могут автоматически подхватываться из папки `операторы/`;
- будние, выходные и праздничные дни различаются автоматически;
- на выходные и праздники используются только `weekend_choices`;
- ставка ограничивает максимальную длительность непрерывной работы: `0.25 -> 4`, `0.5 -> 6`, `0.75 -> 8`, `1.0 -> 8`;
- приложение предоставляет визуальный редактор для ручного назначения сотрудников на смены;
- ручные назначения сохраняются в базе данных и учитываются при расчёте статистики;
- есть возможность экспорта готового графика в Excel.
- учеба и строгие ограничения блокируют назначение на пересекающуюся смену;
- для учебы с указанным временем автоматически добавляется буфер `30 минут` до начала и `30 минут` после окончания;
- планировщик жестко старается не оставлять операторские окна пустыми и не оставлять сотрудников с недобором часов;
- если рабочих интервалов или доступности месяца физически не хватает, приложение показывает это как критическую проблему;
- мягкие пожелания влияют на рейтинг кандидата, но не блокируют его полностью;
- нагрузка распределяется по оставшимся целевым часам и штрафуется за переработки и длинные серии подряд.
