# Управление ограничениями сотрудников

## 📚 Возможности

Система позволяет управлять ограничениями и пожеланиями сотрудников через базу данных с красивым интерфейсом.

### Типы ограничений:

1. **Учёба (строгие ограничения)** ❌
   - Время, когда сотрудник **не может** работать (лекции, пары, экзамены)
   - **Нельзя** ставить работу в это время
   - Пример: `Понедельник 14:00-16:00 — пары`

2. **Пожелания (мягкие ограничения)** 💭
   - Дни, когда сотрудник **хочет** выходной или **хочет** работать
   - **Можно** ставить работу, если график не сходится (в крайних случаях)
   - Пример: `15 марта — хочу выходной (день рождения)`

## 🎯 Как использовать

### 1. Через веб-интерфейс

Откройте в браузере:
```
http://127.0.0.1:8000/constraints/study       → Управление учёбой
http://127.0.0.1:8000/constraints/preferences → Управление пожеланиями
```

#### Добавление учёбы:
1. Выберите сотрудника
2. Укажите дату
3. Укажите время начала и окончания учёбы
4. (Опционально) Добавьте комментарий
5. Нажмите **"Добавить"**

#### Добавление пожелания:
1. Выберите сотрудника
2. Укажите дату
3. Выберите тип: "Хочу выходной" или "Хочу работать"
4. (Опционально) Добавьте комментарий
5. Нажмите **"Сохранить пожелание"**

---

### 2. Импорт из документов сотрудников 📥

Автоматическое извлечение ограничений из Word-файлов сотрудников.

**Вкладка "Импорт из документов":**
1. Укажите папку с документами (например, `операторы/`)
2. Нажмите **"Запустить импорт"**
3. Система проанализирует файлы и сохранит найденные ограничения в БД

**Что извлекается:**
- Учеба (строгие ограничения)
- Пожелания по графику (мягкие ограничения)

## 💾 Структура базы данных

### Таблица `study_constraints`
```sql
CREATE TABLE study_constraints (
    id INTEGER PRIMARY KEY,
    employee_name TEXT NOT NULL,
    date TEXT NOT NULL,
    start_time TEXT NOT NULL,
    end_time TEXT NOT NULL,
    note TEXT DEFAULT '',
    is_strict INTEGER DEFAULT 1,  -- 1 = строгое, 0 = мягкое
    created_at TEXT,
    updated_at TEXT
);
```

### Таблица `schedule_preferences`
```sql
CREATE TABLE schedule_preferences (
    id INTEGER PRIMARY KEY,
    employee_name TEXT NOT NULL,
    date TEXT NOT NULL,
    preference_type TEXT NOT NULL,  -- "prefer_off" или "prefer_work"
    note TEXT DEFAULT '',
    created_at TEXT,
    updated_at TEXT
);
```

## 📊 Как учитывается при построении графика

### Учёба (строгие ограничения):
```python
# Сотрудник НЕ может работать во время учёбы
if constraint.kind == "study" and constraint.is_strict:
    if shift_time overlaps study_time:
        return False  # Запрещено!
```

### Пожелания (мягкие ограничения):
```python
# Сотрудник МОЖЕТ работать, но это нежелательно
if constraint.kind == "prefer_off" and not constraint.strict:
    if shift_time overlaps preference_date:
        score -= 50  # Штраф, но не запрет
        # Назначается только если нет других вариантов
```

## 🧪 Примеры использования

### Пример 1: Добавление учёбы
```
Сотрудник: Иванов
Дата: 15.03.2026
Время: 14:00 - 16:00
Комментарий: Пары в университете
```

**Результат:** Иванов не будет назначен на 15.03 с 14:00 до 16:00.

### Пример 2: Добавление пожелания

Дата: 20.03.2026
Тип: Хочу выходной
Комментарий: День рождения
```

**Результат:** Петрова не будет назначена на 20.03, если есть кто-то другой. Если нет — может быть назначена.

```

**Результат:** Ограничения сохранены в БД для всех сотрудников.

## 🎨 Интерфейс

Интерфейс состоит из 3 вкладок:

1. **Учёба** — таблица с ограничениями по каждому сотруднику
2. **Пожелания** — таблица с пожеланиями
3. **Импорт** — форма для автоматического импорта из документов

Каждая таблица показывает:
- Дату
- Время (для учёбы)
- Тип ограничения
- Комментарий
- Кнопку удаления

---

##  Заметки

- **Учёба** всегда строгое ограничение (`is_strict=1`)
- **Пожелания** всегда мягкие ограничения (`strict=False`)
- Ограничения учитываются **автоматически** при построении графика
- Можно добавлять несколько ограничений на один день
- Учёба с одинаковым временем на один день не дублируется
